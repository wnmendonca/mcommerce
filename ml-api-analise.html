Fase 3 · Expansão

Suprimentos e Estoque

A API do ML fornece dados de estoque em faixas aproximadas. O controle exato, FIFO e reposição inteligente dependem do ERP Tiny.

O que o ML entrega

  • Quantidade disponível em faixas: available_quantity
  • Estoque por variação: GET /items/$ID/variations
  • Pausar anúncio: PUT /items/$ID → qty: 0
  • Retomar anúncio com estoque reposto

O que vem do Tiny

  • Quantidade exata em estoque por SKU
  • Controle de FIFO
  • Gestão de validade de produtos
  • Cálculo de necessidade de compra
  • Histórico de movimentações
Limitação do ML — Estoque por Faixas A API pública do ML não retorna o número exato de unidades em estoque. Retorna faixas: RANGO_1_50 = 1, RANGO_51_100 = 50, RANGO_501_5000 = 500. Para controle real de estoque, o Tiny é obrigatório.

Lacuna Crítica — Mercado Ads não tem API pública Dados de impressões, cliques, CTR, gasto e ROAS do Mercado Ads não são expostos via API documentada.

Alternativas de curto prazo: (1) Entrada manual de dados de campanha no sistema. (2) Importação de CSV exportado do painel do Mercado Ads. (3) Negociação com o ML para acesso a dados — via parceria oficial.

Recomendações de Arquitetura

Decisões técnicas críticas para implementar a integração com o ML de forma robusta e escalável.

01

Camada de ingestão isolada

Separar a coleta de dados da API ML em um serviço independente com fila (BullMQ / Redis). Nunca chamar a API do ML diretamente do dashboard.

02

Banco de dados próprio

Armazenar todos os dados processados localmente (PostgreSQL). Os dashboards leem do banco próprio, nunca das APIs em tempo real.

03

Cache por endpoint

Produtos e comissões mudam pouco — cachear por 1h. Pedidos novos: polling a cada 15min ou via webhook. Evitar rate limit.

04

Multi-conta desde o início

Arquitetura preparada para múltiplos seller_id e múltiplas contas. Cada token OAuth armazenado por conta de vendedor.

05

Cálculo de margem na ingestão

Calcular margem no momento de processar o pedido, cruzando ML + Tiny. Nunca calcular em tempo real na query do dashboard.

06

Monitorar recategorizações

Desde out/2025 o ML recategoriza itens automaticamente. Isso muda comissões e logística. Monitorar o tópico items via webhook.

07

Atenção ao campo price (2026)

Desde mar/2026, requisições atualizando APENAS price são rejeitadas com erro 400 se a automação de preços estiver ativa.

08

Webhooks assíncronos

Responder HTTP 200 em até 500ms. Processar a notificação de forma assíncrona via fila — nunca bloquear o endpoint de webhook.
